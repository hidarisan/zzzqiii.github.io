利用hugo创建个人博客

利用hugo创建个人博客

  1. 安装chocolatey、hugo、git

    https://www.cnblogs.com/kreo/p/4685988.html

    https://gohugo.io/getting-started/installing/

    #用管理员打开PowerShell，首先改变PowerShell策略,允许运行签名脚本

    Set-ExecutionPolicy RemoteSigned

    #然后直接执行代码进行安装:

    Set-ExecutionPolicy Bypass -Scope Process -Force; iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))

    #用Chocolatey直接安装posh-git

    choco install poshgit

    #用Chocolatey直接安装hugo

    choco install hugo -confirm

  2. 新建一个GitHub库

    新建一个 GitHub repository，库名为 username.github.io(后缀一定不能少，惨痛教训），username 即你的 GitHub 账号 username(也很重要)。

  3. 创建Hugo网站

    # 进入想要存放网站文件夹的目录 cd /last/ # 新建一个 Hugo 网站

    hugo new site zblog

  4. 修改主题

    #在zblog中新建一个 themes 的文件夹用于存储主题。

    Hugo Themes: https://themes.gohugo.io

    cd themes
   
git clone https://github.com/g1eny0ung/hugo-theme-dream.git dream

  5. 新建文章

    #进入想要存放网站文件夹的目录 cd .\zblog

    #新建文章

    hugo new post/first.md

  6. 本地预览网站效果

    #启动hugo

    hugo server -D

  7. 构建hugo网站

    在zblog下执行hugo 命令

  8. 将网站文件夹转换为git库

    在public文件夹下执行如下命令

    #初始化git库

    git init

  9. 将git本地库关联至远程库

    在public目录下执行如下命令

    git remote add origin git@github.com:zzzqiii/zzzqiii.github.io .git

  10. 提交修改至本地库

在public下执行以下命令

git status

git add .

git commit -m “Add a new post” #必不可少

  1. 将修改推至远程库

    在public下执行以下命令

    git push -u origin master

  2. 删掉在github上更改站点发布源（必做）

    github默认转到main分支上，在仓库的settings里找到GitHub Pages，把默认的main改为master。

    参考https://docs.github.com/cn/github/working-with-github-pages/configuring-a-publishing-source-for-your-github-pages-site

    附：更改默认分支

    附：about github pages

  3. 发布新文章

    在zblog下执行下面命令新建文章

    hugo new post/paper.md

    更新后

    hugo

    #编辑完后更新GitHub上的内容

    在public目录下

    git status #查看当前修改状态

    git add . #添加所有修改过的文件

    git commit -m “Add a new post” #引号内容是本次提交的说明

    git push -u origin master
